Events

Fabuloos’ events

renderer.changing Fabuloos have to change the renderer.
renderer.ready The renderer finished loading and its API is now ready.
renderer.error An error occurred in the renderer.
renderer.unsupported The renderer the player is trying to use is not supported.
no.renderer No renderer was found for the source the player is trying to load.

Playback events

play The player is trying to launch the playback. It doesn’t means the playback actually started (see playing).
playing The playback has started.
pause The playback has been paused. Either because the pause() method was fired, because the player is buffering (see waiting) or because the payback ended.
waiting The playback has stopped because the next frame is not available.
canplay The player can resume playback, but estimates that if playback were to be started now, the source could not be played to its end without having to stop for further buffering.
canplaythrough The player estimates that if playback were to be started now, the source could played all the way to its end without having to stop for further buffering.
seeking The user asked to seek.
seeked The player seeked to the asked position.
timeupdate The current playback position changed.
ended The playback has stopped because the end of the source was reached.

Network events

loadstart The player started loading the source.
loadeddata The player can render the media at the current playback position for the first time.
loadedmetadata The player has just determined the duration and dimensions of the source.
progress The player is buffering the source.
suspend The player stopped buffering but does not have finished downloading the source.
abort The player stops fetching the media data before it is completely downloaded, but not due to an error.
emptied The source is emptied. Happens when the player starts loading another source.
stalled The player is trying to buffer but data is unexpectedly not forthcoming.

Other events

error An error occurred.
durationchange The duration attribute has just been updated.
volumechange Either the volume attribute or the muted attribute has changed.
